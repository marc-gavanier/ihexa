# Invoice Numbering Rules (Numérotation des factures)

## Legal basis

- CGI: Article 242 nonies A, annexe II
- Sanctions: 15 EUR per incorrect mention

## Fundamental principles

1. **Unique**: each invoice number must be used exactly once
2. **Sequential**: numbers must follow a continuous ascending order
3. **Chronological**: number order must match issuance date order
4. **No gaps**: no missing numbers in the sequence

## Allowed formats

| Format             | Example            | Notes                         |
|--------------------|--------------------|-------------------------------|
| Pure sequential    | 0001, 0002, 0003   | Simplest, always valid        |
| Year prefix        | 2026-001, 2026-002 | Reset counter per fiscal year |
| Year-month prefix  | 2026-05-001        | Reset counter per month       |
| Establishment code | PAR-001, LYN-001   | For multi-site businesses     |
| Mixed              | 2026-PAR-001       | Combines year + site          |

## Resetting the counter

- Allowed at the start of a new fiscal year
- Allowed at the start of each month (if chosen consistently)
- Must NEVER create duplicates across the entire business history
- The prefix (year, month, site) ensures uniqueness

## Distinct numbering series

The tax authority allows separate numbering series when justified by
business conditions:
- Different business units or establishments
- Different activity types
- Different billing systems

Each series must maintain its own continuous sequence.

## Credit notes (avoirs)

- Must follow a **separate** numbering sequence from invoices
- Same rules apply: unique, sequential, chronological, no gaps
- Common format: AV-2026-001 or AVOIR-001

## Common mistakes to avoid

- Deleting an invoice to reuse its number → **illegal**
- Gaps in numbering (even if accidental) → triggers audit suspicion
- Same number used twice → invalidates both invoices
- Non-chronological numbering → fiscal irregularity
